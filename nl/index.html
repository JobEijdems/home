Zowel back-end als front-end

Je data moet niet alleen worden opgeslagen; het moet worden gebruikt. Wij specialiseren in het ontwikkelen van interactieve linked data systemen, waarin gebruikers items kunnen aanmaken en bewerken. Onze frontend applicaties passen zich automatisch aan aan wijzigingen in het data model.

We hebben vele jaren ervaring met het bouwen van eenvoudig te gebruiker webapplicaties die werken op linked data back-ends.

Hoe wij werken

  1. 1. Luisteren en begrijpen

    Goed advieswerk begint met aandachtig luisteren. Samen met jou kijken we naar de databronnen en bespreken we hoe ze worden gebruikt. Wij denken mee over hoe jouw doel bereikt kan worden. Zowel kleine pilot projecten als grote overstaptrajecten naar een full-stack linked data warehouse zijn mogelijk.

  2. 2. Vind of ontwerp een ontologie

    Bij linked data is standaardiseren belangrijk. Door gebruik te maken van een bestaand datamodel, ontologie, vocabulair of taxonomie, wordt jouw data beter herbruikbaar. Daarom helpen we bij het vinden van een bestaande standaard die werkt met jouw data. Als dat niet lukt, helpen we graag bij het ontwerpen en maken van een passende ontologie. We gebruiken internationale W3C standaarden, zoals SHACL, RDFs, SKOS en OWL om jouw ontologie vorm te geven.

  3. 3. Extract, transform, load

    We kunnen een ETL architectuur opzetten die data importeert uit diverse bestaande systemen. Dit kunnen data dumps zijn, bestaande API's van applicaties, of iets anders. We zetten alles om naar Linked Data. Onze ETL structuur werkt op basis van 'events', zodat we direct een meldingen dienst kunnen opzetten.

  4. 4. Software ontwikkeling

    We bouwen systemen die het gemakkelijk maken om jouw linked data weer te geven en aan te passen. Of je een eenvoudige web interface of een specifieke machine interface nodig hebt: wij kunnen het. We zijn ervaren full stack developers die zowel backends als frontends ontwikkelen. We hebben onder andere ervaring met Ruby (on Rails), Javascript, Node, Typescript, Python, HTML, CSS, Docker en Continuous Integration.

  5. 5. Publiceren

    Al jouw data kunnen we beschikbaar stellen middels een RESTful Hypermedia API die serializeert naar JSON, JSON-LD, N3, N-Triples, CSV, RDF/XML en meer. We kunnen alles doorzoekbaar maken en beschikbaar maken via Cypher, SPARQL, GRAPHQL en ElasticSearch. Jij kunt instellen wie wat mag bekijken of bewerken.
