Verbind jouw datamodellen

Een greep uit onze projecten

Open Raadsinformatie

Open Raadsinformatie is een applicatie die vergader- en besluitinformatie verzamelt uit software van meer dan 100 Nederlandse gemeenten. Dit wordt vervolgens toegankelijk gemaakt als linked data via een API en een zoekmachine. Open Raadsinformatie hebben wij samen met de VNG en de Open State Foundation gerealiseerd.

Link-Lib / Link-Redux

Link is een door ons ontwikkelde verzameling van open source libraries voor front-end webapplicaties. Ze helpen bij het binnenhalen, opslaan, toegankelijk maken en het weergeven van linked data. De ingebouwde reasoner helpt bij het bepalen van de beste weergave voor nieuwe type data.

RDF Serializers

Wij hebben een open source library voor Ruby on Rails ontwikkelt om linked data mee te serialiseren. Deze library ondersteunt RDF/XML, Turtle, N3, N-triples en JSON-LD.

Argu

Argu is een e-democracy platform dat besluitvorming transparanter en interactiever maakt. Het is ontworpen voor het voeren van oplossingsgerichte en genuanceerde discussies, makkelijk en veilig stemmen en online brainstormen. Alle data op Argu is toegankelijk als linked data middels een intuïtieve REST API.

Hoe wij werken

  1. 1. Luisteren en begrijpen

    Goed advieswerk begint met aandachtig luisteren. Samen met jou kijken we naar de databronnen en bespreken we hoe ze worden gebruikt. Wij denken mee over hoe jouw doel bereikt kan worden. Zowel kleine pilot projecten als grote transities naar een full-stack linked data warehouse zijn mogelijk.

  2. 2. Vind of ontwerp een ontologie

    Bij linked data is standaardiseren belangrijk. Door gebruik te maken van een bestaand datamodel, ontologie, vocabulair of taxonomie, wordt jouw data beter herbruikbaar. Daarom helpen we bij het vinden van een bestaande standaard die werkt met jouw data. Als dat niet lukt, helpen we graag bij het ontwerpen en maken van een passende ontologie. We gebruiken internationale W3C standaarden, zoals SHACL, RDFs, SKOS en OWL om jouw ontologie vorm te geven.

  3. 3. Extract, transform, load

    We kunnen een ETL architectuur opzetten die data importeert uit diverse bestaande systemen. Dit kunnen data dumps zijn, bestaande API's van applicaties, of iets anders. We zetten alles om naar Linked Data. Onze ETL structuur werkt op basis van 'events', zodat we direct een notification service kunnen opzetten.

  4. 4. Software ontwikkeling

    We bouwen systemen die het gemakkelijk maken om jouw linked data weer te geven en aan te passen. Of je een eenvoudige webinterface of een specifieke machine interface nodig hebt: wij kunnen het. We zijn ervaren full stack developers die zowel back-ends als front-ends ontwikkelen. We hebben onder andere ervaring met Ruby (on Rails), Javascript, Node, Typescript, Python, HTML, CSS, Docker en het opzetten van een OTAP straat.

  5. 5. Publiceren

    Al jouw data kunnen we beschikbaar stellen middels een RESTful Hypermedia API die serializeert naar JSON, JSON-LD, N3, N-Triples, CSV, RDF/XML en meer. We kunnen alles doorzoekbaar maken en beschikbaar maken via Cypher, SPARQL, GRAPHQL en ElasticSearch. Jij kunt instellen wie wat mag bekijken of bewerken.
