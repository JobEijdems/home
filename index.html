How we work:

  1. 1. Understand your data

    We will look at your data, discuss about what it represents and how it is manipulated.

  2. 2. Find the right model

    Linked data is all about standardization. Using an existing model or standard makes your data more reusable, so we will help you find existing ontologies that might work for your data. If that does not exist, we’ll help you with Ontology design.

  3. 3. Extract, load, transform

    We’ll set up an architecture that imports your data, whether that’s a data dump, some specific API or anything else. Then, all data will be transformed to linked data.

  4. 4. Publish

    All your content can be made accessible through a RESTful API that serializes to JSON, JSON-LD, N3, N-Triples, CSV, RDF/XML We can add endpoints for Cypher, SPARQL and GRAPHQL queries You set read / write permissions, we make sure only those who should, can access your data.
