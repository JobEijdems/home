How we work

We love technological challenges. Although every project is unique, we tend to solve challenges in a similar manner:

  1. 1. Listen & understand

    Together with you, we will look at your data sources and discuss what they represent and how they are used. We will identify your most valuable information, evaluate how we can tackle your challenge and decide on how to advance.

  2. 2. Find or design an ontology

    Linked data is all about standardization. Using an existing model or standard makes your data more reusable, so we will help you find existing ontologies that might work for your data. If such an open model does not exist, we can help you design a fitting ontology.

  3. 3. Extract, transform, load

    We’ll set up an architecture that imports your data, whether that’s a data dump, a specific API or anything else. Then we will set up the ETL processes to transform your data to linked data.

  4. 4. Publish

    All your content can be made accessible and editable through a RESTful Hypermedia API that serializes to JSON, JSON-LD, N3, N-Triples, CSV, RDF/XML and more. We can make everything searchable and queryable with Cypher, SPARQL, GRAPHQL and ElasticSearch. You set read / write permissions, we make sure only the right people and systems can access your data.

  5. 5. Build

    Data is rarely static. That’s why we build systems that make it really easy to modify and view your linked data. Whether you need an intuitive web interface or a custom made machine interface: we’ve got you covered.
